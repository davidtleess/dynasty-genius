3

Hyperparameter Optimization & Validation

Alpha Search — RidgeCV per Position

Replace the static alpha=100.0 with per-position RidgeCV. Each of the 4 models searches independently for the alpha that balances fit and generalization against its own position's sample size and noise profile.

QBs may favor higher regularization (smaller N). WRs have the largest dataset and may tolerate lower alpha. TE alpha is fully independent of WR despite shared features.

Stratified Validation Report

Each training run produces a per-position validation_report.json containing a three-way comparison:

  • v1.0 Baseline — original unified model (historical reference)
  • v1.1 Control — unified model minus route_participation
  • v2.0 Target — stratified position-specific model

All three rows reported per position for clean attribution.

Model Architecture Route Participation Stratified Role

Promotion Gate — per position

A v2.0 artifact is promoted only if it beats the naive baseline (prior-year PPG) on ≥ 2 of 3 metrics: RMSE, R², Spearman rank correlation — the same gate used for Engine B v1.0. Per-position decisions are independent. A failing position is not promoted; its v1.0 counterpart remains active with appropriate caveats.

4

TE Promotion Rules & Experimental Caveat

TE is currently marked ENGINE_B_EXPERIMENTAL_POSITIONS = {"TE"} because v1.0 does not beat the naive baseline. This status is evidence-gated. It is not cleared by design decision, architectural change alone, or assumption.

If te_v2.pkl passes gate (≥ 2/3 metrics)
  • Promote te_v2.pkl to production
  • Remove TE from ENGINE_B_EXPERIMENTAL_POSITIONS
  • Update service layer and roster auditor display
  • Log promotion evidence in validation report and ledger
If te_v2.pkl fails gate (< 2/3 metrics)
  • te_v2.pkl is not promoted
  • TE remains in ENGINE_B_EXPERIMENTAL_POSITIONS
  • v1.0 TE artifact remains active with existing caveat
  • Failure is logged as Phase 6.2 evidence for future TE diagnosis
Hard rule: No agent may remove TE from ENGINE_B_EXPERIMENTAL_POSITIONS without a passing validation report in the ledger. The TE experimental caveat propagates through the service layer, API response, and all roster auditor display surfaces until cleared by evidence.

RB Feature Enrichment — Phase 6 Follow-On

Deferred: RB Usage Signal Expansion

RBs in Phase 6 run on the Foundational + Historical + Aging feature set only. No new RB-specific usage metrics are added in the first stratification pass. This keeps the experiment clean: Phase 6 answers "Does stratification improve Engine B on the current validated feature universe?" It does not simultaneously answer "What RB features are missing?"

After the stratified RB model benchmarks, if R² or Spearman underperforms relative to the WR model, the following are the prioritized Phase 6 follow-on candidates. All require a source coverage check, leakage check, and validation gate before promotion.

red_zone_touches targets_per_game weighted_opportunity_rb carries_per_game goal_line_touches

Note: carries_per_game is likely highly correlated with snap_share for RBs and may not add independent signal. red_zone_touches is the highest-priority candidate for PPR dynasty value.
