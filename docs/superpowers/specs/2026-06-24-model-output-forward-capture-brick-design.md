## 0. Compounding-product lens
- **Daily-login value:** a dated model-value series enables descriptive *model trend*; joined at analysis time with the FC market series it seeds the descriptive model-vs-market divergence-over-time view.
- **Refresh cadence:** **two distinct cadences** — the **PVO refresh** re-scores daily as a pulse-check (Option C, §7a); the **capture** appends a daily observation (D1). New *vintages* accrue only when `semantic_output_hash`/`provenance_hash` change (real input/model change); identical daily re-scores are classified honestly as `vintage_changed=false` (no new vintage) — the daily observation is still appended per D1, **not** dedup-suppressed (C15).
- **Compounding:** capture-and-accumulate, append-only, never overwrite; the gitignored capture `.db` is the immutable compounding record.

## 1. Scope (TWO parts, both in scope per D2)
- **A — Capture brick:** read the **published** model-output artifact (`app/data/valuation/universe_pvo_latest.json` + `universe_pvo_coverage_latest.json`) into a dedicated append-only, survivorship-complete, PIT store + a machine-readable capture report. **Model-only — market fields excluded.**
- **B — PVO-refresh scheduler (Option C, §7a):** a runner that **re-scores in place locally** to keep the published artifact current, **never auto-commits**, **Phase-17.2 PVO-rebuild only** (not the full league-intelligence chain), **zero** feature/training/model mutation.

## 2. Capture contract (ARTIFACT-READ)
- Source: read the published `universe_pvo_latest.json` (12,201 rows; top-level `captured_at`/`coverage`/`schema_version`/`source_snapshot_captured_at`/`players`) + coverage sibling. **No re-score in the capture path** (the §7a refresh re-scores; capture only reads what's published).
- Source family: `model_pvo` (`assert_single_source_family`).
- Fail-loud: missing/malformed/empty artifact → `aborted`, NO write. **Required-provenance-missing for any model-supported row → abort the whole run.** Idempotent (§3) so retry is safe.

## 3. Storage contract (append-only, survivorship-complete, PIT)
- Dedicated parallel store `app/data/model_forward_capture.db` (gitignored — §6), **NOT** the FC namespace. Tables: `model_forward_capture_raw` (survivorship-complete — **every** one of the 12,201 rows, incl identity-unresolved/non-model, null model provenance) + `model_forward_capture_joinable` (model-supported, resolved, Sleeper-keyed, for the future divergence join).
- **Model-supported row (C11):** `valuation.engine_path ∈ {ENGINE_A, ENGINE_B, BLEND_AB}` (current: ENGINE_A=80, ENGINE_B=373 → 453). All other paths (`PRE_MODEL`=9607/`INACTIVE`=2140/`UNRESOLVED_IDENTITY`=1/future `CONTEXT_ONLY`/`MARKET_ONLY`) → raw-sidecar only, null provenance, never abort.
- **Append-only / immutable**: never overwritten/recomputed.
- **Composite key (vintage = Codex 3-hash):** `(capture_date, source='model_pvo', semantic_output_hash, provenance_hash, player_key)`. **Vintage is defined by `semantic_output_hash` + `provenance_hash`, NOT the literal `artifact_sha256`** (which changes every refresh via volatile timestamps/run-IDs even when scores are identical — `artifact_sha256` is an **audit** field in §4, not a key). `dvs_engine`/`engine_path` are data, not key.
- **`player_key` fallback (C12/C13, no collisions):** `sleeper:<identity_ids.sleeper_id|sleeper_player_id>` only when present, non-blank, **and not `'0'`** → else `dg:<dg_player_id>` when non-null/non-blank → else `unresolved:<semantic_output_hash>:<row_index>:<semantic_row_hash>`. **The unresolved key uses STABLE semantic hashes, never the volatile `artifact_sha256`** (C13): `semantic_row_hash` = per-row content hash **excluding** volatile `captured_at`/`assembled_at`/`pipeline_run_id`, so an unresolved row stays the same key across same-content re-runs (preserving idempotency/duplicate detection).
- **Idempotency:** identical same-key payload → no-op; changed value at an existing key → conflict hard-fail; duplicate `player_key` within one artifact → hard-fail unless byte-identical.

## 4. Provenance / vintage block (resolved READ-ONLY at capture)
Three distinct hashes (Codex): **`artifact_sha256`** (literal audit hash of the generated file — changes every refresh); **`semantic_output_hash`** (canonical hash of the model-output rows **excluding** volatile `captured_at`/`assembled_at`/`pipeline_run_id`/run-IDs — the actual scored content); **`provenance_hash`** (input/model/config lineage). **Real changed vintage ⇔ `semantic_output_hash` and/or `provenance_hash` changes.** Plus:
- **Artifact:** `pvo_artifact_path` + `artifact_sha256` + `artifact_vintage` (**= top-level `captured_at`**) + `source_snapshot_captured_at` + `pvo_schema_version`; `coverage_artifact_path` + `coverage_sha256`.
- **Timing (stale-honesty):** `capture_date` (today, UTC) — distinct from `artifact_vintage` — + `artifact_age_days`.
- **Repo/producer:** `git_sha`; `pvo_producer` path/hash (`build_universe_pvo_batch.py`); per-row `lineage.sleeper_snapshot_hash` + `lineage.governance_version` + `pipeline_run_id`.
- **Engine B:** `engine_b_manifest` (`v2_manifest.json`) path/hash + per-position artifact path/hash/version/features_hash; `engine_b_derived_training_cutoff = 2023` **labeled `derived`** from `engine_b_features_v2.csv` (max training-eligible season; inference 2024) + `feature_csv` path/hash/max_training_season.
- **Engine A (C2/C8):** **both** `engine_a_v2_pointer` (`app/data/models/latest.json` model_version/run_dir/hash) **and** `head_a_v3_manifest` hash + `te_v3_metadata` path/hash (TE rows); `engine_a_training_cutoff` as its **own** per-engine status (not the Engine-B value; `unknown`/`derived` marker if unresolved).
- **`provenance_hash` canonicalization:** `sha256` over canonical JSON (sorted keys, UTF-8, repo-relative paths, explicit `null`s) of the **lineage subset**: artifact schema/`source_snapshot_captured_at`, `pvo_producer` hash, engine_b manifest/per-position/feature hashes + derived cutoff, engine_a pointers/hashes + cutoff. **Excludes `artifact_sha256`, `capture_date`, `artifact_age_days`, volatile artifact timestamps, AND `git_sha` (C14)** — `git_sha` is stamped as **audit metadata only**; including repo HEAD would falsely change the vintage on an unrelated/docs-only commit. (If commit-lineage in the vintage is ever wanted, use a scoped `producer_git_sha`, not full HEAD.) `semantic_output_hash` similarly excludes volatile fields.
- **Unresolved markers / abort:** unresolvable required provenance for any model-supported row → **abort whole run**; non-model rows persist with null provenance.

## 5. Capture report (every run)
`status`, `capture_date`, `artifact_vintage`, `artifact_age_days`, `raw_rows`, `joinable_rows`, `counts_by_engine_path`, `missing_sleeper_count`, `unresolved_count`, `duplicate_count`, **`market_fields_excluded_count`** (rows with a non-null `market_overlay`/`divergence` excluded), `artifact_sha256`, `semantic_output_hash`, `provenance_hash`, `vintage_changed` (vs last captured), `store_hash`, the §4 block, `aborted_reason`, and **`decision_supported=false`**. Latest-overwrite report ok (operational audit); the **store is the canonical record**.

## 6. Guardrails
**Capture:** read-only sidecar; market fields excluded (recursive contract test fails on `market_overlay`/`divergence` keys); descriptive-only; divergence UNVALIDATED until Gate-4; `decision_supported=false`; banned tokens zero-tolerance; `artifact_age_days` prevents stale-as-fresh. **Refresh (Option C):** re-runs the **existing** producer (Phase-17.2 PVO rebuild) only — **zero** mutation of models/`.pkl`/features/training/the Engine math path; **never auto-commits** (no-scheduler-commits discipline); a **dirty working tree** for the regenerated `universe_pvo_latest.json`/coverage is **expected local operational state, not a failure**. **Gitignore:** `app/data/model_forward_capture.db`(+`-journal`), the capture-report dir `app/data/model_capture/`, refresh-report dir, and `app/data/logs/model_forward_capture.*` / `app/data/logs/pvo_refresh.*`. The committed `universe_pvo_latest.json`/coverage stay **tracked** (load-bearing for 4 API routes); committing a refreshed baseline is a **David-gated** action.

## 7a. PVO-refresh runner (Option C — Codex deps 1–5)
A runner that: (1) mutates **only** `universe_pvo_latest.json` + `universe_pvo_coverage_latest.json` via the **Phase-17.2 PVO rebuild** (NOT the full league-intelligence chain; David must explicitly expand to run more); (2) **backup/restore** around the two tracked artifacts + a **refresh report** with pre/post `artifact_sha256` + `semantic_output_hash` + `provenance_hash`, `dirty_paths`, `semantic_changed`/`provenance_changed` flags, `commit_required_for_repo_baseline=true`, `decision_supported=false`; (3) in the orchestrated path **refresh → then capture**, but the **capture command stays independently callable + daily-capture-capable** even if refresh is skipped/failed; (4) **never** runs `assemble_engine_b_dataset.py` / `train_engine_b.py` / any `.pkl`-producing command; (5) state docs distinguish **local-runtime freshness** from **committed-repo-baseline freshness**.

## 9. Falsification matrix seeds
**Capture:** missing/malformed/empty artifact → abort; required-provenance unresolvable for a model-supported row → abort; non-model row → raw sidecar null-provenance, never abort; same-day identical re-run → no-op; same-day changed value at existing key → conflict hard-fail; duplicate `player_key` → hard-fail unless byte-identical; identity-unresolved row (`sleeper='0'`,`dg=null`) → `unresolved:<…>` key, never `sleeper:0`/`dg:null`; any `market_overlay`/`divergence` key in an emitted model row/report → reject; `semantic_output_hash`/`provenance_hash` **stable** across two `capture_date`s for identical scores+lineage (canonicalization test) and **change** when scores/lineage change; `artifact_sha256` may differ while `semantic_output_hash` is unchanged → `vintage_changed=false`; `engine_b_derived_training_cutoff` not labeled `derived`/applied to Engine-A rows → fail; report missing any required field → fail; banned token in any emitted field/report/static string → fail. **Refresh:** runner attempts a commit → fail (no-scheduler-commits); runner runs anything beyond Phase-17.2 / touches `.pkl`/features/training → fail; backup not restored on refresh failure → fail; refresh report missing `dirty_paths`/`commit_required_for_repo_baseline`/pre-post hashes → fail; refresh on unchanged inputs → `semantic_changed=false`/`provenance_changed=false` (honest no-new-vintage).
