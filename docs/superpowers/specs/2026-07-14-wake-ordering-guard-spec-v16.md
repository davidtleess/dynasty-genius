## G1 — Retry loop (unchanged since r7; carried whole)
Canonical `target_snapshot_date` (YYYY-MM-DD from aware-UTC at start). Classify every read: ==target FRESH · <target RETRYABLE (only) · >target FATAL · missing/corrupt FATAL. Config `WINDOW=1800`/`INTERVAL=60`, exact non-bool positive ints, INTERVAL ≤ WINDOW. Per wake: UTC-rollover check → monotonic deadline check (≥ ⇒ FINAL read) → read+classify → clipped sleep from the POST-read monotonic sample. Post-read UTC/monotonic resample decides finality and the date law in either wall-jump direction; a read returning past midnight terminates; a read past deadline IS final, classified normally. Publish-time UTC gate before the first artifact write. `retries_used` = reads after the first.

## G3 — Commit model
Generation directory `generations/<commit_seq>_<generation_id>/` holds latest+coverage (INERT until referenced). **Publication order: history transaction → artifact files → iff ok: COMMITTED marker → ATTEMPT marker; degraded: ATTEMPT marker only. Reports write after ALL markers, or not at all.**
- Markers: two independently stable config-pinned paths. Committed = ok-only serving truth; attempt = every handled terminal. **Two ordinals (r15-2):** `attempt_seq` (strictly monotonic across ALL runs: prior attempt marker's +1, or the committed marker's +1 if higher, or 1) persists in every attempt marker; the committed marker records the `attempt_seq` of the run that produced it, plus `commit_seq` (prior committed +1).
- **Committed-rename outcome (r15-3):** ok path = attempt the COMMITTED rename FIRST; success ⇒ write the ok ATTEMPT marker; **failure ⇒ write a DEGRADED attempt marker (terminal_classification=publish_failed)** — a known publish failure is never silent. If THAT attempt write also fails: log + exit nonzero (the unavoidable gap; covered by cadence).
- **Marker-pair reduction (exhaustive over attempt_seq):** attempt.attempt_seq == committed.attempt_seq ⇒ normal. attempt.attempt_seq > committed.attempt_seq ∧ attempt degraded ⇒ current failure (last-good serves under the dash law). attempt.attempt_seq < committed.attempt_seq ⇒ attempt_lagged warning (crash after commit before attempt — incl. a prior failed run's marker still standing; benign, self-heals). attempt marker ABSENT ⇒ attempt_missing warning (first-run crash; cadence governs). attempt.attempt_seq > committed ∧ attempt ok ⇒ impossible under the pinned order; surfaced as integrity alarm, committed serves. Every state is in the closed sets; nothing is "impossible" without a defined observed-behavior.
- **Reports are AUDIT-ONLY and OUTSIDE producer terminal truth (r14-1):** written last, named by attempt_id, mirroring the terminal markers; a report-write failure is LOG + nonzero exit ONLY — it is not a marker state, not a terminal_classification, and no marker references any report field. Report consumers validate by attempt_id + the shared validator against the markers.
- GC: whole-generation quarantine = one rename to `.trash/`; ordering = the dirname seq; retention = the current-committed directory + anything younger than GRACE_GC(24h) mtime; placement in the total order: GC runs at owning start, BEFORE the new run's history transaction, under the same lock; GC failure = log + nonzero, run continues (pruning is never load-bearing). Partial deletion RESUMES from `.trash/` (entries there are deletion roots — the sweep re-walks and completes them; nothing is ever re-quarantined from `.trash`, r15-12). **Historical attempt reports are CURRENT-ONLY:** a report validates only against the standing markers; older reports are archival text, declared so. Reader ENOENT ⇒ discard the whole snapshot, re-read committed + both companions once; second miss fails close.

## G4 — Verified loader + closed status law
- Loader: shared strict validator FIRST (malformed ⇒ marker_malformed) → status precedence → for ok: open the generation dir's files, recompute both hashes from returned bytes (mandatory), **then STRICT-PARSE the served bytes semantically: duplicate-key rejection, numeric bounds, and recursive exact-false on every decision_supported (root and nested) — reject-true, never coerce (r15-10; the current Pulse/Trade coercers are BF-11). The same law applies to every generated DTO the callers emit.**
- **ONE closed lane-status enum:** `ok` · `ok_stale` · `no_committed_generation` · `legacy_unguarded` · `evidence_lost` · `marker_malformed` · `generation_mismatch` · `output_hash_mismatch` · `companion_unreadable` · `producer_failure` (status value; `reason` is a separate field — never spliced) · `gap_basis_stale` · `pvo_regressed`. **Warnings (`lane_warnings[]`):** `pvo_pending_lineage` · `pvo_content_anomaly` · `attempt_lagged` · `attempt_missing` · `chain_mixed`. **`clock_anomaly` is NOT a warning — it forces `ok_stale`** (r15-4/5: warnings NEVER change caller behavior; anything that must change rendering is a status).
- **`ok_stale` = the RATIFIED DASH LAW (r15-4):** the primary surface renders NO market values — market columns dash exactly as composition v3.14:374 requires; the prior vintage and last values live in the DISCLOSURE/RECEIPT only. Trade divergence context requires status == ok exactly (not ok_stale, not warnings-clean).
- **Precedence (pinned, first match wins):** marker_malformed > generation_mismatch > output_hash_mismatch > companion_unreadable > producer_failure > pvo_regressed > gap_basis_stale > clock/schedule staleness (ok_stale) > ok. Absence states (no_committed_generation / legacy_unguarded / evidence_lost) are structural and precede all of the above. **legacy_unguarded serves unsuffixed market numerics BY EXCEPTION (flagged)** — the "numerics only under ok/ok_stale" rule reads "ok / legacy_unguarded" after this round (ok_stale no longer serves numerics). Field-level nullability follows the status table.
- **PVO branches (exhaustive; status first, warnings second):** PVO unresolvable/unverified ⇒ companion_unreadable. Bytes EQUAL ⇒ ok (+pvo_pending_lineage warning iff lineage differs). Bytes DIFFER ∧ current lineage OLDER ⇒ pvo_regressed. Bytes DIFFER ∧ lineage same ⇒ gap_basis_stale + pvo_content_anomaly warning. Bytes DIFFER ∧ lineage newer ⇒ gap_basis_stale.
- **Cadence (ONE authority — r14-3):** `wake_ordering_guard.json` carries `schedule` ("09:40 America/New_York daily") and `grace_minutes` (30). `report_freshness.json`'s legacy `grace_hours` is MIGRATED to reference this config (a build item with an equality test binding loader, health, and the launchd plist). `due = next occurrence after committed_finished_at`; `overdue = now > due + grace` ⇒ ok_stale (FULL dash; vintage in the receipt). Degraded-and-newer attempt ⇒ ok_stale immediately. `committed_finished_at > now` ⇒ ok_stale (clock case). **#148 replacement schema (executable, r15-11):** `{"verification_mode": "marker_committed", "committed_marker": <path>, "attempt_marker": <path>, "schedule_ref": "wake_ordering_guard.json"}` replaces the single-path + grace_hours entry (a named migration with an equality test). **Timezone self-check:** at start the runner compares its host-local rendering of the configured schedule against the config timezone; mismatch ⇒ fatal_config (a host tz change cannot silently shift the cadence).
- **Caller table (state × behavior):** players: HTTP 200; market-lane numerics null except under ok/ok_stale; `market_lane_status` + `lane_warnings` + `as_of {pair, pvo, price_fetch?}`; ok_stale renders values + dash styling + age. trade combined route: pricing per G6-price; divergence context only under ok (not ok_stale — a stale gap never renders as current context); `divergence_status` carries the enum. divergence-only endpoints: 503 {error, status} for non-ok/non-ok_stale. health: the enum value + warnings as named bases. Pulse/opportunity: per-section (G8). Manager copy per state is a build-RED table seeded from these semantics.

## G7 — History (exact-set, authenticated, computable authority)
- One transaction: validate candidates (nonempty unique normalized string IDs; strict-parsed payloads — duplicate keys rejected, non-finite rejected, numerics bounded; SQL `decision_supported` = exact integer 0; the payload's own decision_supported fields = exact JSON false, recursively) → DELETE(day) → INSERT full set → upsert `history_day_meta` {capture_date TEXT PK, generation_id TEXT|NULL, content_sha256 TEXT(64), row_count INT, written_at_utc TEXT, legacy BOOL, **surface_committed BOOL default false**}.
- **ORDER RECUT (r15-6): the history transaction runs AFTER the committed marker** (publication order becomes: artifact files → committed → attempt → HISTORY transaction → report). Consequences, all pinned: a failed run never touches history (surface A keeps history A — the destroy-A defect is gone); a crash after commit before history leaves surface B with history A(pending) — **start-time reconcile:** every owning start compares the committed marker's day+generation to `history_day_meta`; on mismatch it REBUILDS that day's history from the committed generation's own artifact bytes (the data is in the committed latest file — fully computable recovery, no same-day rerun needed). `surface_committed=true` is written IN the history transaction itself (which now runs post-commit, so it states a fact, not a hope); pending = false on a non-legacy day; historical committed days keep true permanently.
- content_sha256 = MANIFEST V3 (the landed, cross-verified algorithm; byte rules as G2). Readers recompute count+hash in ONE snapshot; mismatch or missing-meta-post-effective ⇒ fail-close. Legacy (pre-effective) days readable, flagged. Evidence extractor status: strict-parse-everywhere + universe validation + one-transaction reads LANDED (sha ae643112…).
