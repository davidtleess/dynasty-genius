## 0. Compounding-product lens
- **Daily-login value:** the model's view of players updates as the season plays — without the user remembering to do anything (David's Q2 challenge).
- **Refresh cadence:** event-driven — regenerate ONLY when upstream `nflreadpy` data actually changed (honest `noop` otherwise); auto-refresh when 2026-season data lands.
- **Compounding:** each fresh feature vintage is a new PIT model stance; the longer it runs, the richer the model-vs-market divergence series toward Gate-4.

## 1. Goal / non-goals
- **Goal:** a source-hash-gated feature-refresh that regenerates the engine_b feature set from current `nflreadpy` data, re-scores with the FROZEN v2 models, and produces distinct vintages — safely, unattended, and honestly labeled.
- **Non-goal (Q1):** NO model retraining. Model weights stay frozen; retrain is a separate, future, human-gated phase. The refresh path is **physically barred** from `.fit()`, training entrypoints, and writing any `.pkl`/model artifact.
- **Non-goal:** NOT a divergence-edge validation. Refreshing features does NOT validate the edge; `decision_supported=false` unchanged; **Gate-4 stays a separate forward study.** Vintage movement is mechanical accrual, not a signal.
- **Non-goal:** market never becomes a feature (constitutional; re-verified every refresh).

## 2. Decisions — CLOSED (Q1–Q4, 3-opinion cockpit + David-confirmed)
- **Q1 — feature-refresh ONLY** (frozen weights; `.fit()`/training/`.pkl`-write barred + audit test; retrain-ready provenance hooks, never invoked). Rationale: **isolation of variables** — to ever evaluate the model's edge (Gate-4), the math must stay frozen while only the data moves.
- **Q2 — automated source-hash-gated NO-COMMIT scheduler is the END STATE** (David's challenge: manual-only = staleness risk), sequenced AFTER the safety gates. A manual run is the first-run/catch-up, not the end state. Honest `noop` in the offseason; `as_of`/vintage freshness labeling surfaced downstream.
- **Q3 — SEED-SPLIT** (resolves F-seed-split for features): a committed **seed** CSV (durable, reproducible, David-gated audit baseline) + a **gitignored runtime** CSV the refresh writes. Scoring/provenance read runtime-else-seed and stamp WHICH.
- **Q4 — INTEGRITY gates block; DRIFT is report-only telemetry.** Gate on structural validity + leak-freeness, never on movement (movement is the goal). Fail-closed + disclosed.

## 3. Architecture / components
- **Feature-refresh runner** (`scripts/run_feature_refresh.py`): regenerates the feature set from current `nflreadpy` data with the latest available season **preserved as the inference partition** (C1, §4). Source-hash-gated; idempotent; `--preflight` readiness-only; `main(argv)->int`; cwd-independent bootstrap. Writes a temp CANDIDATE only.
- **Inference-partition refactor (C1):** the current `scripts/assemble_engine_b_dataset.py` (lines ~352–358) computes the outcome, sets `training_eligible = feature_season < 2024`, then **drops rows with a null outcome** — so a latest/in-progress season (no T+1/T+2 outcome yet) is dropped before it can be scored. The refactor must **preserve intended-inference-season rows with `training_eligible=false` and a null outcome ALLOWED**, while training rows still require a complete outcome. The **inference-season rule** is defined explicitly (the max completed source season / current-season boundary, not a hardcoded `< 2024`), with a test.
- **Validation gate module** (integrity gate; reuses `engine_b_contract` + the `assemble_engine_b_dataset` leakage check): runs all §5 gates on the candidate; fail-closed.
- **One shared resolved-feature-source helper (C2):** a single helper resolves the active feature CSV (runtime-if-present-else-seed) + returns its path/hash/source-kind, and is the ONLY feature-CSV reader used by **every** current consumer — `app/services/engine_b_service.py` (`_DATASET_PATH`), `scripts/build_universe_pvo_batch.py` (`ENGINE_B_FEATURES_PATH` — note it BOTH reads feature rows itself AND calls `score_inference_partition()`; BOTH must route through the helper so PVO rows can never mix predictions from one CSV with features/provenance from another), and `src/dynasty_genius/capture/model_forward_capture_driver.py` (`ENGINE_B_FEATURE_CSV_PATH`). Tests prove runtime-present uses the runtime hash/path in ALL three.
- **Scheduler (final task, after gates green)**: a LaunchAgent wrapping the runner; source-hash-gated; NO-COMMIT (Option-C); atomic + lock-file + reader-side ready-marker (C6, §4); scheduled before the 09:30 PVO window; honest `noop`.

## 4. Seed-split + invocation contract
- **Seed:** the current `app/data/training/engine_b_features_v2.csv` becomes the committed seed (durable baseline; changes ONLY on a David-gated `runtime → seed` promotion).
- **Runtime:** a gitignored runtime feature CSV + its provenance/report (added to `.gitignore`). The runner writes the runtime via **atomic temp-candidate → validate → atomic rename**; never in-place.
- **Resolution:** scoring/PVO/capture resolve **runtime-if-present-else-seed**. **Fail-closed:** if a runtime CSV exists but FAILS validation, abort/refuse to score and preserve/restore the prior valid runtime — **NO silent fallback to seed** (fallback only when runtime is ABSENT).
- **Source-hash input set (C4):** the source hash is computed over a DEFINED, canonicalized input set — the `nflreadpy` loader outputs + the seasons window, the `nflreadpy`/package version (if resolvable), the builder config/constants, the TE rubric/eligible artifacts consumed by `add_te_role_risk_feature_from_files`, and any identity/crosswalk inputs if included — and EXCLUDES generation timestamps/wall-clock. Ambiguity here causes false `noop`s or false vintages, so the set is enumerated + tested.
- **Reader-side safety (C6):** atomic rename protects the CSV bytes, but the feature CSV + validation/provenance/report bundle could be observed mid-publication. The resolver consumes a **validation/ready manifest marker** (only a fully-validated, fully-published bundle is resolvable) — or the PVO refresh participates in the same advisory lock before resolving runtime — so a reader sees old-valid-or-new-valid, never partial.
- **Runner status semantics:** `status=ok, refresh_performed=true` when source hashes changed and the runtime regenerated; `status=noop, refresh_performed=false, source_hash_unchanged=true` when nothing changed; `status=blocked` + failed-gate reason when validation fails. Every report stamps `decision_supported=false`, source hashes, output artifact hashes, `dirty_paths`, `commit_required_for_repo_baseline` (true only when tracked artifacts changed), and `runtime_promotable_to_seed` (+hashes) for David's later gated seed publication.
- **No auto-commit ever** (Option-C / no-scheduler-commits). The scheduler regenerates on-disk runtime only; committing seed baselines is exclusively David-gated.

## 5. Validation gates (Q4 — fail-closed acceptance; drift report-only)
1. **Leakage gate** — rerun `MARKET_PROHIBITED` / `engine_b_contract` on every candidate (constitutional; market never a feature).
2. **Schema gate** — exact required columns, no market-like columns, strict dtype coercion, stable key columns.
3. **Identity/key gate** — no blank IDs on scored rows, no duplicate player-season rows, valid position values.
4. **Coverage gate** — total row floor + per-position floors + **inference-season presence** (abort if the intended refreshed season is absent).
5. **Range sanity** — rates/shares ∈ [0,1], plausible age, non-negative counts, plausible season values.
6. **NaN integrity** — no all-null critical features, per-feature NaN ceilings, position-specific allowed-null rules.
7. **Drift telemetry (REPORT-ONLY)** — row-count / NaN-rate / distribution deltas vs prior runtime for review; **never blocks** (movement is the goal).
8. **Disclosure** — validation status + failure reason surface in the refresh report AND downstream API/digest freshness metadata; a failure refuses scoring and alerts David (surfaced status on login), never silently serves stale/invalid runtime.

## 6. Provenance / vintage + freshness labeling
- Provenance/vintage MUST reflect the RESOLVED feature CSV actually scored (runtime-if-present, else seed — C7): stamp `feature_source_kind` (runtime|seed), `feature_csv_path`, `feature_csv_sha256`, `feature_source_hash`, `source_as_of`/`generated_at`, `published_seed_sha256`. A genuine feature change moves `feature_csv_sha256` → `provenance_hash` → and `semantic_output_hash` when scores differ.
- **Provenance vs audit boundary (C5):** `generated_at`/report time is **audit-only and EXCLUDED from `provenance_hash`**; `source_as_of` is included in provenance ONLY if it is derived from source CONTENT (e.g. the latest source data date), never from wall-clock execution time. **False-vintage guard:** provenance changes ONLY from real source/validated-output hash changes — never from timestamps or `noop` runs.
- **Freshness labeling (Q2):** the league API + the What-Changed model section surface the model `as_of`/vintage date so a long offseason `noop` is VISIBLE — never imply false daily freshness.

## 7. Guardrails (carried from all four decisions)
Feature-refresh only (frozen weights; `.fit()`/training/`.pkl`-write barred + audit test); market never a feature (leakage gate every refresh); `decision_supported=false`; divergence UNVALIDATED (Gate-4 separate); seed-split no-auto-commit (Option-C / no-scheduler-commits); fail-closed on invalid runtime; atomic writes + backup/restore + advisory lock; honest `noop` + `as_of` labeling; frontend HOLD.

## 8. Build sequence (cockpit TDD: Codex RED → Claude GREEN → dual-CLEAR → David-authorized commit)
**Re-sequenced (C3): production scoring must NOT be able to read a runtime CSV until the validation gates exist — so the resolver/downstream wiring lands AFTER the gates, never in T1. T1 split (cockpit-cleared Option B): the full feature-engineering extraction is its own task T1b, BEFORE T2.**
1. **T1 — runner + schema-conformant SEAM candidate (no production read):** `scripts/run_feature_refresh.py` (source-hash-gated regeneration to a temp CANDIDATE only; `--preflight`; status semantics; cwd-independent) + the **C1 inference-partition refactor** (preserve the single latest inference season `training_eligible=false`/null-outcome-allowed; explicit `inference_season_rule`; training rows still require complete outcomes) + the legacy `assemble_engine_b_dataset` outcome/drop → shared-rule refactor (no `<2024`/`dropna`) + the gitignored runtime/provenance/report paths + the **no-`.fit()`/no-model-write/no-training-entrypoint audit test**. The candidate is a **schema-conformant SEAM** (exact `ENGINE_B_OUTPUT_COLUMNS`; feature values needing full engineering may be null pre-T1b — NOT yet scoreable); the **CLI real-run is GATED** so no seam artifact publishes before T1b. NO resolver, NO production read of runtime, no scheduler.
2. **T1b — frame-injectable full feature-engineering extraction (before T2):** extract `assemble_engine_b_dataset`'s 11-step engineering into `feature_assembly.build_engine_b_features(*, read_fns, seasons_window)` so `assemble_feature_candidate` produces a **genuinely scoreable** candidate with REAL feature values; `assemble_engine_b_dataset` becomes a thin loader delegating to it (module-level names + committed output preserved for the 4 dependents + `run_pvo_refresh`); T1b RED verifies REAL feature values (snap/route/pbp/QB-eff/aging/TE fixtures); removes the T1 real-run gate.
3. **T2 — validation gates + atomic publish (validates ONLY the T1b real candidate):** the §5 integrity gate module (fail-closed) + atomic temp→validate→rename (candidate→runtime) + ready-marker/manifest (C6) + preserve/restore-on-failure + drift report-only telemetry + report disclosure. Only a fully-validated bundle becomes resolvable.
4. **T3 — shared resolver + downstream wiring (C2):** the single shared resolved-feature-source helper, routed through ALL consumers (`engine_b_service`, `build_universe_pvo_batch` [both its direct feature-row read AND `score_inference_partition`], `model_forward_capture_driver`); provenance stamps the source-kind/hash; the league API + What-Changed model section surface the `as_of`/vintage freshness label; reader-side ready-marker honored. **One-time latest-complete-season (2025) catch-up = the first real manual run here, David-gated — produces the first non-flat vintage.**
5. **T4 — automated scheduler (after the gates are green):** LaunchAgent wrapping the runner; source-hash-gated; NO-COMMIT (Option-C); atomic + lock-file; scheduled before the 09:30 PVO window; honest `noop`. launchctl go-live David-gated/separate.

## 9. Falsification matrix seeds
Refresh path invokes `.fit()` / a training script / writes under the model-artifact dir → audit test FAILS. Upstream `nflreadpy` unchanged → `noop` (no write, no vintage change, no false provenance). Candidate has a market-prohibited column → leakage gate fail-closed. Candidate missing the intended inference season / below row-or-position floors → coverage gate fail-closed. Runtime exists but fails validation → abort + restore prior valid runtime (NO silent seed fallback). Runtime absent → seed fallback (stamped `feature_source_kind=seed`). Torn CSV byte-read impossible via atomic rename; partial-bundle resolution refused without the ready-marker (C6). Provenance stamps the SEED hash while RUNTIME was scored → test FAILS. `noop`/timestamp changes the vintage → test FAILS. Scheduler auto-commits a tracked artifact → FAILS (no-commit). Digest/API omits the model `as_of` → FAILS (freshness labeling). **(C1)** regeneration with the latest season available produces NO inference rows for that season (assembler dropped them) → test FAILS; training rows still require a complete outcome. **(C2)** any consumer (`engine_b_service` / `build_universe_pvo_batch` direct-read / `build_universe_pvo_batch` `score_inference_partition` / `model_forward_capture_driver`) reads the feature CSV NOT via the shared resolver → test FAILS; PVO combines feature-rows from one CSV with predictions/provenance from another → test FAILS. **(C4)** identical source data on different runs → identical source-hash (`noop`); a real upstream data change → source-hash changes. **(C5)** `generated_at`/wall-clock appears in `provenance_hash` → test FAILS. **(C6)** resolver consumes a partially-published bundle lacking the ready-marker → must refuse to resolve it.
