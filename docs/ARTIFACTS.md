# Dynasty Genius Artifact Index

All generated artifacts are immutable once written. Do not hand-edit artifact files; regenerate them with the listed script so provenance, schema validation, and gate semantics stay intact.

Market-derived artifacts are trust and comparison layers only. They must not be joined into Engine A or Engine B training features.

## Trust Surface API

Routes live in [`app/api/routes/trust_surface.py`](../app/api/routes/trust_surface.py).

| Route | Reads | Behavior |
|---|---|---|
| `GET /api/trust-surface/{position}` | latest `backtest_result_{POS}.json` | Returns the raw `BacktestResult` plus top-level `overall_grade`, `experimental`, and `model_card_available` |
| `GET /api/trust-surface/{position}/model-card` | `{POS}_model_card.json` | Returns the persisted `ModelCard` JSON or 404 if no card exists |

## Draft Pick Valuation Artifacts (Phase 24)

Locations: `app/data/valuation/` (curve) and `app/data/backtest/phase24/` (calibration).

| File | Schema | Generated by | Informs |
|---|---|---|---|
| `draft_pick_value_curve_v1.json` | slot/tier curve: `version`, `board_size`, `mature_years_used`, `slots`, `tier_map`, `tiers`, `source` | [`scripts/build_draft_pick_value_curve.py`](../scripts/build_draft_pick_value_curve.py) | Dynasty rookie-pick xVAR in the Trade Lab evaluator and `team_value_matrix` (future picks are valued but **excluded** from starter strength) |
| `sf_qb_knob_calibration_<ts>.json` | calibration result: `recommended_k`, `median_raw`, `n_drafts`, `n_qbs_matched`, `n_qbs_unmatched`, `per_draft`, `promotions`, `classes`, `caveats` | [`scripts/calibrate_sf_qb_knob.py`](../scripts/calibrate_sf_qb_knob.py) | The SF-QB ordering knob `sf_qb_promote_slots` (K); gated on David's explicit approval of the computed K before any curve regen |

How the curve is built (provenance carried in its `source` block):

- **Bridge:** the first-36 skill players in NFL draft order ≈ a 36-slot FF rookie board; realized `y24_ppg` → DVS → xVAR per historical slot, over `mature_years_used` (2015–2022).
- **Option A option-value floor:** per-player `priced_xvar = max(0, raw_xvar)`, aggregated by **mean** — a pick is a call option, so busts floor at 0 rather than dragging the slot negative. Then a monotonic clamp + median tiers (`round_N_generic` for round-only resolution).
- The curve is an **NFL-derived historical expectation, not a market-measured price**. All pick values carry `decision_supported=False`.

SF-QB knob status (calibration of 2026-05-26): **K = 0** — median promotion `0.0` across 6 real SF rookie drafts / 27 matched QBs (3 unmatched, classes 2022/2024/2025/2026). `sf_qb_promote_slots` therefore stays `0`: the curve's "FF order ≈ NFL skill order" assumption holds for QBs in this (thin) corpus. The knob is re-runnable as the league accrues drafts; the artifact carries `sf_qb_calibration_thin_sample`.

## Operational Commands

Run the current artifact pipeline in this order:

```bash
.venv/bin/python3.14 scripts/run_backtest.py --all
.venv/bin/python3.14 scripts/run_backtest.py --position TE
.venv/bin/python3.14 scripts/generate_model_cards.py --all
.venv/bin/python3.14 scripts/build_divergence_ledger.py --all
```

Draft-pick valuation artifacts are regenerated on demand (not part of the backtest pipeline):

```bash
.venv/bin/python3.14 scripts/build_draft_pick_value_curve.py   # rebuild draft_pick_value_curve_v1.json
.venv/bin/python3.14 scripts/calibrate_sf_qb_knob.py           # recompute the SF-QB knob K (read-only Sleeper)
```

Use `--market-store app/data/fc_snapshots.db` on `run_backtest.py` when a valid snapshot store is available.

## Harness Trust Completion — Gate-4 forward-collection clock (W2a)

- **Daily `fc_native` snapshot cadence.** `scripts/snapshot_fantasycalc.py` runs **daily** (~09:00 local) via the LaunchAgent `ops/launchd/com.davidleess.dynasty-fc-snapshot.plist`, writing immutable FantasyCalc snapshot rows (`source=fc_native`) to `app/data/fc_snapshots.db`. Local-first; no Databricks.
- **Immutability contract.** Writes go through `MarketSnapshotStore.append_snapshots` (verify-or-raise): a same-`(snapshot_date, league_settings_hash, sleeper_id)` re-run with identical values is an idempotent no-op; a *changed* value for an already-recorded date raises `MarketSnapshotImmutabilityError` (fail loud — no silent overwrite). This protects point-in-time provenance.
- **Manual activation (David).** The plist is committed but loaded manually: `launchctl load ~/Library/LaunchAgents/com.davidleess.dynasty-fc-snapshot.plist`. The agent never auto-loads it.
- **Gate-4 readiness clock.** Gate 4 (divergence forward-return validity) needs a forward-collected, point-in-time snapshot series that does not exist yet. **Gate-4 readiness date = first-collection-date + ~6 months** (a six-month forward window). W2b — the `DivergenceResult` producer + matched-control design (spec §8.3) — stays deferred until this ~6-month readiness window matures. Every day the clock is not running pushes Gate-4 out a day.
- Market/FantasyCalc data here remain **overlays only** — never an Engine A/B feature/training input; `decision_supported=False` throughout.
