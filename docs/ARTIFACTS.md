# Dynasty Genius Artifact Index

All generated artifacts are immutable once written. Do not hand-edit artifact files; regenerate them with the listed script so provenance, schema validation, and gate semantics stay intact.

Market-derived artifacts are trust and comparison layers only. They must not be joined into Engine A or Engine B training features.

## Backtest Run Artifacts

Location: `app/data/backtest/runs/{run_id}/`

| File | Schema | Generated by | Informs |
|---|---|---|---|
| `backtest_result_{POS}.json` | `BacktestResult` in [`backtest_artifact.py`](../src/dynasty_genius/eval/backtest_artifact.py) | [`scripts/run_backtest.py`](../scripts/run_backtest.py) | G1 rank correlation, G2 RMSE stability, G3 market superiority, G4 divergence readiness |
| `predictions_{POS}.csv` | `PREDICTION_LOG_COLUMNS` in [`backtest_report.py`](../src/dynasty_genius/eval/backtest_report.py) | [`scripts/run_backtest.py`](../scripts/run_backtest.py) | Calibration, ECE, subgroup slices, model cards |
| `market_comparison_{POS}.json` | `list[MarketComparisonEntry]` in [`backtest_report.py`](../src/dynasty_genius/eval/backtest_report.py) | [`scripts/run_backtest.py`](../scripts/run_backtest.py) with optional `--market-store` | G3 model-vs-market comparison, divergence ledger substrate |

Positions: `QB`, `RB`, `WR`, `TE`.

Operational note: the first Phase 12 run artifacts were generated before the prediction and market sidecar files existed. Re-run `scripts/run_backtest.py` with current code to produce the full run directory shape.

## Model Card Artifacts

Location: `app/data/backtest/model_cards/`

| File | Schema | Generated by | Informs |
|---|---|---|---|
| `{POS}_model_card.json` | `ModelCard` in [`model_card.py`](../src/dynasty_genius/eval/model_card.py) | [`scripts/generate_model_cards.py`](../scripts/generate_model_cards.py) | Trust Surface v2, model transparency, TE diagnosis |
| `{POS}_calibration_report.json` | `CalibrationReport` in [`model_card.py`](../src/dynasty_genius/eval/model_card.py) | [`scripts/generate_model_cards.py`](../scripts/generate_model_cards.py) | ECE review, calibration diagnostics, DVS readiness review |

Model cards follow the 9-section Mitchell-style structure implemented in `ModelCard`. TE cards must keep `is_experimental = true` and must include known failure modes.

## Divergence Ledger

Location: `app/data/backtest/`

| File | Schema | Generated by | Informs |
|---|---|---|---|
| `divergence_ledger_{POS}.json` | `list[DivergenceLedgerEntry]` in [`backtest_report.py`](../src/dynasty_genius/eval/backtest_report.py) | [`scripts/build_divergence_ledger.py`](../scripts/build_divergence_ledger.py) | Passive model-vs-market disagreement review, future G4 substrate |

This is not yet a decision signal. It records model rank vs. market rank disagreement for later validation once enough native FantasyCalc snapshots exist.

## Market Snapshot Store

Location: `app/data/fc_snapshots.db` (gitignored)

| Asset | Schema | Written by | Informs |
|---|---|---|---|
| FantasyCalc snapshot SQLite store | `MarketSnapshotStore` in [`market_snapshot_store.py`](../src/dynasty_genius/eval/market_snapshot_store.py) | [`scripts/snapshot_fantasycalc.py`](../scripts/snapshot_fantasycalc.py) and archive ingestion scripts | G3 NDCG comparison, future G4 divergence validity |

The store is optional for backtest runs. If it is absent, market comparison fields remain unavailable and generated market-comparison artifacts may be empty lists.

## Trust Surface API

Routes live in [`app/api/routes/trust_surface.py`](../app/api/routes/trust_surface.py).

| Route | Reads | Behavior |
|---|---|---|
| `GET /api/trust-surface/{position}` | latest `backtest_result_{POS}.json` | Returns the raw `BacktestResult` plus top-level `overall_grade`, `experimental`, and `model_card_available` |
| `GET /api/trust-surface/{position}/model-card` | `{POS}_model_card.json` | Returns the persisted `ModelCard` JSON or 404 if no card exists |

Trust Surface routes are read-only. They must not run the backtest harness, generate model cards, build divergence ledgers, mutate artifacts, or trigger model fitting.

## Operational Commands

Run the current artifact pipeline in this order:

```bash
.venv/bin/python3.14 scripts/run_backtest.py --all
.venv/bin/python3.14 scripts/run_backtest.py --position TE
.venv/bin/python3.14 scripts/generate_model_cards.py --all
.venv/bin/python3.14 scripts/build_divergence_ledger.py --all
```

Use `--market-store app/data/fc_snapshots.db` on `run_backtest.py` when a valid snapshot store is available.
