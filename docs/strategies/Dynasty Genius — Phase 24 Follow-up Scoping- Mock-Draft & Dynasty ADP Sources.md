# Dynasty Genius — Phase 24 Follow-up Scoping: Mock-Draft & Dynasty ADP Sources

## TL;DR
- **Follow-up B (Dynasty Rookie ADP ingestion) is more feasible to do first.** MyFantasyLeague's public, documented `/export?TYPE=adp` endpoint (with `IS_KEEPER=Rookie Only`, `FCOUNT=12`, `IS_PPR=1`, `IS_MOCK=No`) is real-draft-derived, settings-filterable, free, and historically deep — it slots directly into the existing `MarketSource` abstraction with minimal added complexity. Sleeper's `GET /draft/<draft_id>/picks` endpoint gives a complementary, also-free per-draft signal. FantasyCalc already covers the year-round veteran/rookie market.
- **Follow-up A (near-class NFL mock-draft aggregation) is feasible but tier-1 sourcing is gated.** Three credible aggregation paths exist: (i) the NFL Mock Draft Database API (rich, machine-readable, but priced for builders and licensed against redistribution), (ii) NFL.com's stable of named analysts (Jeremiah/Zierlein/Brooks/Edholm/Parr/Reuter — free but bespoke HTML), and (iii) Grinding the Mocks (Benjamin Robinson's R Shiny dashboard with 2018–2026 EDP coverage — methodologically strongest but distributed as a viz, not a feed). A historical backtest of projected→realized capital is feasible back to ~2018 via Grinding the Mocks and back to 2003 via WalterFootball's archive (confirmed: walterfootball.com/nfldrafthistory.php lists 2003 NFL Mock Draft, 2004, 2005 … 2026 in unbroken sequence; the 2003 Round 1–3 file walterfootball.com/draft2003_3.php was last updated 4/24/2003).
- **Two hard red flags:** PFF's Sandbox/API license is "limited, revocable, non-exclusive, non-sublicensable, non-transferable" and forbids derivative works outside testing; NFL Mock Draft Database's API explicitly "prohibit[s] redistribution, white-labeling, and building a competing aggregator" with a per-response watermark. Both are unsuitable as primary feeds for a redistributable product; both remain viable for **private, internal, non-redistributed** inference-time overlays only.

## Key Findings

**Mock-draft landscape.** The mock-draft space has consolidated around two true aggregators (NFL Mock Draft Database; Grinding the Mocks) plus a long tail of single-analyst boards. NFL Mock Draft Database aggregates 1,500 expert-authored mocks, 200 big boards, and 1,500 team mocks per season into a daily consensus, with an explicit JSON API; its API page states "eight years of curated aggregation across 850+ sources" and "All data from 2017 to today." The Wide Left consensus-board comparison piece independently confirms NFLMDDB is one of the two largest aggregators alongside Grinding the Mocks. Grinding the Mocks publishes a Bayesian "Expected Draft Position" (EDP) metric with selectable draft years 2018–2026 on its public dashboard; the dashboard surfaces quoted endorsements including the Wall Street Journal ("might just be the most accurate tool to make sense of…the NFL draft") and The Ringer ("It's one of the best available tools to approximate basic expectations heading into a draft") — both outlets hedge ("might just be" / "one of the best") rather than crowning it definitively. For Engine A's "projected NFL draft capital range" the EDP is conceptually the closest match because it is already an expected-pick projection with uncertainty bands, not just an arithmetic mean.

**Aggregation feasibility.** Bleacher Report's published 2026 methodology — averaging the latest mocks from six named experts (The Ringer / Diante Lee, The Athletic / Dane Brugler, NFL.com / Daniel Jeremiah, ESPN / Mel Kiper, ESPN / Field Yates, and the B/R Scouting Department of Brandon Thorn, Dame Parson, Daniel Harms, and Matt Holder) and assigning the average to a projected slot — is a workable manual blueprint for a 3+ source consensus. Daniel Jeremiah's NFL.com author page exposes a clean per-version history (Mock 1.0, 2.0, 3.0, Final) per season, and Lance Zierlein, Bucky Brooks, Eric Edholm, Dan Parr, and Chad Reuter all publish on the same nfl.com URL pattern (`/news/{analyst}-{year}-nfl-mock-draft-{version}`).

**Dynasty ADP landscape.** Real-draft-derived rookie ADP is concentrated in three places: (i) **MFL** itself, queryable via a documented public API with parameters that exactly match the user's league shape (`IS_KEEPER=Rookie Only`, `FCOUNT=12`, `IS_PPR=1`); (ii) **DLF's MFL Rookie ADP** page, which is a curated re-presentation of MFL data behind the DLF Premium paywall and is seasonality-limited (spring only, when actual MFL rookie drafts run); and (iii) **Sleeper** draft pick data via `GET /draft/<draft_id>/picks`, which is per-draft and free but requires you to enumerate league IDs you already know about. Dynasty Data Lab (cited by FantasyPros as its 2026 SF rookie ADP source) is real-Sleeper-draft-derived but is a one-person SPA with no documented API, ~2024-onward history, and no published redistribution terms (operator @Adeiko_FF Twitter account created Feb 2024; first Sleeper-ADP reference July 2024).

**Mocks vs. real drafts within "ADP" feeds.** DLF's Superflex Rookie ADP is described on its own page as "drawn from Superflex Rookie-Only dynasty **mock drafts** organized by @RyanMc23"; DLF's MFL Rookie ADP is "gathered from actual rookie drafts in dynasty leagues on MyFantasyLeague.com." DynastyNerds rookie ADP is editorially **projected** by staff after each NFL Draft day, not crowd-derived. FantasyPros consensus rookie ADP aggregates MFL, BB10, Underdog, Sleeper, Drafters, and FastDraft — a usable secondary aggregate but mixed across formats.

**KeepTradeCut posture.** KTC explicitly states in its FAQ: "We don't currenly have an API or any sort of .csv available for our rankings and values data." KTC is also crowdsourced trade-value, not ADP. It is not a candidate for clean programmatic ingestion.

**Settings filterability.** Only MFL's public API exposes simultaneous settings filters (PPR scoring, team count, keeper/rookie-only, mock-exclusion) in URL parameters. Sleeper drafts are per-draft (you bring the settings via the draft you choose). FantasyCalc (already integrated) exposes `isDynasty`, `numQbs`, `numTeams`, `ppr`. KTC has Superflex on/off and TE-premium on/off as UI toggles but no API. FantasyPros consensus rookie ADP is not parameter-filterable to 12T/SF/Full-PPR.

### What's actually free vs. paid, condensed

- **Free, machine-readable, license-OK:** MFL public API; Sleeper API; FantasyCalc (already integrated); WalterFootball pages (own mocks, no aggregator); NFL.com per-author HTML pages; Fantasy Football Calculator REST.
- **Free, viewer-only (data extraction is gray):** Grinding the Mocks Shiny dashboard; Drafttek; FantasyPros consensus pages; KTC.
- **Paid + license-restrictive:** NFL Mock Draft Database API ($5,000 Scout / $50,000 Front Office, redistribution forbidden); PFF Sandbox API (testing-only license); ESPN, The Athletic, DynastyNerds, DLF Premium (all subscriber-only).

## Recommendations

### Follow-up A — 3+ source mock aggregation, recommended composition

**Recommended set:** (1) **Daniel Jeremiah / NFL.com** (free; clean per-version author page), (2) **Lance Zierlein / NFL.com** (free; same URL pattern; second NFL.com voice gives intra-publisher diversity), (3) **Dane Brugler / The Athletic** (paid but high-signal; user has already indicated willingness to subscribe), and (4) **Grinding the Mocks EDP** as a consensus benchmark (treated as a meta-source — never one of the "3+" but always cross-checked). WalterFootball can be added as the fourth named source for free, and is the only candidate with an archive index dating to 2003. If the user wants a fifth pure-analyst voice, **Mel Kiper / ESPN** is the strongest paid add. Avoid PFF as a primary feed for license reasons. For each prospect, blend by simple mean of latest available projected pick, attach a min/max range across the 3+ sources, and surface Grinding the Mocks EDP as a quality check. Computed projected NFL draft capital is then bucketed into rounds (1.early / 1.mid / 1.late / 2 / 3 / 4–7 / UDFA). **A backtest is feasible** for at least the 2018–2025 classes by retrieving final-version mocks from NFL.com author pages, WalterFootball's 2003+ archive, and Grinding the Mocks' 2018+ year selector, joining to actual NFL draft outcomes (publicly available via PFR/nflverse), and computing mean absolute pick error and round-bucket accuracy. The backtest cannot use NFLMDDB or PFF historical data without a license.

### Follow-up B — Recommended ADP feed and integration sketch

**Recommended primary feed: the MyFantasyLeague public ADP API.** Query path: `https://api.myfantasyleague.com/{year}/export?TYPE=adp&PERIOD=RECENT&FCOUNT=12&IS_PPR=1&IS_KEEPER=Rookie+Only&IS_MOCK=No&JSON=1` for in-season rookie ADP; switch `IS_KEEPER` to `No` for veteran/startup ADP; pull `/{year}/reports?R=ADP&L={league_id}` for per-league drill-down to seed the SF-QB calibration corpus. Settings exactly match three of the four binding constraints (12T, Full-PPR, rookie-only); QB-count and TEP cannot be excluded at the API level, but the user's prior finding that DLF MFL ADP is the recommended real-draft proxy already accepts this skew. **Recommended secondary feed: Sleeper draft picks** via `GET /draft/<draft_id>/picks` for any known dynasty league/draft IDs — this gives per-draft granularity (each pick stamped with `picked_by`, `round`, `pick_no`, `metadata.player_id`) that is exactly the shape needed to thicken the SF-QB calibration corpus. **Integration sketch via the existing `MarketSource` abstraction:** add `MflMarketSource` and `SleeperDraftMarketSource` as new concrete implementations alongside FantasyCalc. Each adapter (i) constructs the parameterized URL from a fixed `{numTeams=12, ppr=1, dynasty=true, qbs=2}` config block, (ii) calls the existing 3-stage cache, (iii) applies the existing banned-field sanitizer, (iv) emits the standard `MarketRow{player_id, market_rank, market_value_or_pick, source, fetched_at}` rows with `decision_supported=False` and a `source_kind="real_draft_adp" | "per_draft_picks"` tag, and (v) for SF-QB calibration only, the Sleeper adapter exposes per-draft `(draft_id, pick_no, player_id)` triples through a separate non-market read path. **Do not** introduce DynastyNerds, KTC, DLF SF Rookie ADP (mocks), or Dynasty Data Lab as primary feeds for B — each fails on at least one of license/access/history/real-vs-mock.

### Net recommendation — which follow-up to do first

**Build B first.** B has a free, documented, settings-matched, history-rich primary source (MFL API, ~18 years yearly snapshots) and a free, documented, per-draft secondary source (Sleeper) that together solve both the market-overlay and SF-QB-calibration use cases the user originally scoped. A clean B implementation is roughly two `MarketSource` subclasses with adapter-level tests and zero new license risk. A meets a higher decision bar to start: it requires either (i) accepting NFLMDDB's annual API cost ($5,000/yr Scout tier minimum) and redistribution restrictions, (ii) building a multi-site scraper of 3+ individual analyst HTML pages with brittle CSS selectors and per-source naming normalization, or (iii) hand-curating mocks weekly during draft season. Of those, (ii) is what the practice memo implies, but it is at least 3–5× the engineering cost of B, and its primary deliverable (projected draft capital range) is only meaningful during a 4–5-month window per year (December–April), versus B's year-round signal. The backtest gating on A also requires investment in a normalized prospect-identity table that does not yet exist, whereas B can lean on the existing Sleeper/MFL `player_id` cross-walk that the app already uses.

## Caveats

- The 10-year-history bar is only cleanly met by **MFL** (verifiable yearly ADP back to 2007), **WalterFootball own mocks** (confirmed unbroken 2003→2026 index sequence), and **NFLMDDB** ("All data from 2017 to today" — ~8 years, near-miss). Grinding the Mocks (2018–2026, ~8–9 years) is a stretch hit. Dynasty Data Lab (~2 years) clearly fails. Sleeper (beta launch 2017, ~9 years platform-wide; rookie ADP retention as a derived series is bring-your-own).
- **NFLMDDB pricing correction:** an earlier reading of the API page conflated the two tiers. The correct mapping per the published API page is **Scout $5,000/yr (24-hour delay) for fantasy platforms / dynasty tools / agencies** and **Front Office $50,000/yr (real-time) for NFL front offices, sportsbooks, and large media operations**. For a single-user dynasty app, only Scout is plausibly proportional, and even Scout's redistribution clause and watermark make it usable only as an internal inference-time overlay.
- "Brugler's Beast" is referenced from third-party context; the exact mock format (one final mock vs. ranges-only) was not pulled in this pass. Recommend confirming format before assuming it can be parsed as a "projected pick per prospect" feed.
- DLF's labeling has shifted over the years (per the Mizelle third-party archive, 2007–2011 was "consolidated ADP from a few real drafts"; 2012–onward shifted to "DLF Mock Drafts ADP"). The current DLF distinction between **MFL Rookie ADP = real** and **Rookie ADP = mock** is a more recent (post-2020) editorial split. Backtests using DLF historical pages must be careful about which series they are consuming.
- TE-premium and QB-count filtering are **not** exposed by MFL's API; MFL TEP and 1QB leagues will be silently included in the aggregate. The user's league is 12T/SF/non-TEP. The contamination is bounded and well-understood from prior findings; no mitigation required for an overlay, but flag in source metadata.
- FantasyPros' 2026 consensus mock draft page currently shows "We've compiled 0 mock drafts" — i.e., FantasyPros' own consensus is **not a viable source for the in-season 2027 cycle right now**; only their consensus *rookie ADP* (which blends MFL/Sleeper/etc.) is populated.
- The WSJ and Ringer endorsements on the Grinding the Mocks landing page hedge their language ("might just be the most accurate" / "one of the best"); treat the dashboard as a strong but not definitively-best aggregator, not an oracle.
- All recommendations preserve the constitutional bar: market data is price discovery, not truth; no market-derived feature in model training; all surfaced rows carry `decision_supported=False`. Nothing in this matrix implies lifting any HOLD or NOISE_BAND lock.
