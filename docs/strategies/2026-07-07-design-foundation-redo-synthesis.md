> failure. Four independent inputs (two zero-context synthesis agents over the existing

## The root-cause thesis (all four seats agree)

The foundation is **constraint-complete but aspiration-absent.** The rich research corpus
(the July-5 world-class brief, the DynastyNerds visual benchmark, the deep-research
synthesis, the July-6 fundamentals research, the fresh-agent reviews) contained BOTH a
guardrail layer and a destination layer. The distillation into `PRODUCT.md`/`DESIGN.md`
**kept the guardrails and dropped the destination.** Two lines did the damage:

- `PRODUCT.md`: *"The jaw-drop is credibility, not chrome."* — a true statement weaponized
  into "do not pursue visible beauty"; defines the only permitted jaw-drop as an abstract
  quality you cannot see.
- `DESIGN.md`: *"Signature elements (spend boldness here only)… Everything else stays
  quiet."* — caps boldness at three *honesty mechanisms* (Hard Right Edge, receipts, tape),
  none a moment of visual delight, and forbids craft on the rest of the surface.

**The polarity flip:** honesty is the **substrate**, not the aesthetic. Fantasy-native
category legibility (Sleeper / DynastyNerds / KTC bar) is the aesthetic. Today's foundation
makes honesty *the aesthetic* (DESIGN principle 1); the redo makes honesty *the substrate*
and the fantasy-native asset terminal *the surface.* This is the single largest missing frame.

## The aspirational destination (Agent 1, over the July-5 corpus)

Concrete, translatable, and — proven by the frame-check below — all frame-compatible:

- **Two-pane compare-then-inspect desk** (already DG's daily-open shape): league-wide
  comparison left, selected inspector right. A spatial signature, not a card grid.
- **The value number IS the hero** — big, weighted, focal, one number owns the row;
  everything else is subordinate typography. DG's `ValueHero` is currently undersized vs
  the destination. *(Most-dropped single idea.)*
- **Density done beautifully** — `identity → focal value → uncertainty → trend → chrome` in
  one row; 4/8/12px on a 4px grid, right-aligned tabular numerals, human dates. Density
  through type weight, not lines and boxes.
- **Player-identity richness** — circular headshots, team-color chips in REAL team colors,
  position chips, college logos on prospects. "Data feels human." (DG ships initials-only.)
- **Inline sparklines + net movement number** per row, terminating at the Hard Right Edge.
- **Per-row uncertainty σ bar** with a numeric label — a place DG *exceeds* the benchmark
  (fold CIs, not ranker disagreement).
- **Tier / value-band dividers** with labeled group headers carrying **group total + league
  rank** ("QUARTERBACKS (5) — 10,140 (5/12)").
- **Signature hero moments** — player-highlight hero card (photo + measurables + stat
  chips), graded attribute bars, and **Franchise Equity** (roster value + owned valued picks
  + equity trend) — the franchise-equity spectacle lives in David's own league.
- **Motion-as-meaning** — Carbon two-tier; expressive rationed to the one orchestrated
  daily-open entrance; staged, Congruence-bounded chart transitions.
- **Typographic craft** — three-font weight-based hierarchy (Archivo display / Plex Sans /
  Plex Mono tabular). Drama from scale + weight, which keeps the blue/amber lanes clean.

## The fundamentals + failure learnings (Agent 2, over the July-6 corpus)

**Rankings-first fundamentals:** the canonical row is a fixed grammar
(`rank · position-rank chip · identity · ONE focal value · named-window trend · status
chips`); one number owns the row (2–3× label size); current value + delta always travel
together; rows never begin with caveats; a rank without a disclosed basis is a defect; the
mandatory identity fallback chain (headshot → initials-on-position-color → silhouette, never
a broken image or raw id); evidence rides NEXT TO the number, not only behind hover;
"our view vs market view" is an industry-standard column (DG's core axis has mainstream
precedent); rankings are the primitive every other surface composes from.

**Why the built UI scored 1–5/10 (fresh agents, verbatim convergence):** "an engineer left
the admin view on." The tells: diagnostics/System-Health panel owns the hero; raw database
IDs in ENTERED/EXITED ("the single most credibility-destroying element"); unstyled
caveat wallpaper (law satisfied *literally* — show everything — instead of designed);
sticky-header scroll-collision shipping-blocker; no product shell/wordmark, internal chrome
("DEVELOPER", "(Parked)") leaking into nav; the two-lane system invisible (blue spent on
nav, model card has no blue); undifferentiated rows (no rank, "+109 of what?", repeated
labels); debug-grade identical sparklines; broken mobile (1/10, ~7000px, horizontal
overflow). **Why the cockpit missed it:** full-page screenshots flatten sticky elements;
contract-green stood in for design quality; anchoring (graded the diff, not the screen);
"out of scope" let shell chrome escape every review.

### Candidate principle backbone for the new foundation (P1–P12)

- **P1 — Fantasy-native first, governed underneath.** Visible grammar = asset terminal in
  the Sleeper/DN/KTC category; honesty is the substrate, not the surface aesthetic. *(The
  reframe.)*
- **P2 — The canonical row is a fixed contract.** The row grammar as law; one number owns
  the row; current value + delta travel together; no repeated per-row labels; rank needs a
  disclosed basis.
- **P3 — Player identity is required recognition infrastructure**, not decoration —
  headshots + fallback chain; team = abbreviation + color mark (ring/chip, never row-fill,
  never logo).
- **P4 — The scaffolding-hide law (new hard principle).** No implementation artifact reaches
  the surface: no raw IDs, snake_case, ISO timestamps, diagnostics in the hero, or dev chrome
  in nav. Every quiet/pending/failure state is a *designed* state.
- **P5 — Honesty is designed, not dumped ("visible, not wallpaper").** Receipts/caveats/
  Hard-Right-Edge stay, rendered as designed elements; uncertainty display becomes DG's
  signature honesty asset.
- **P6 — The two-lane system must be visibly a system.** Blue frames model objects, amber
  frames market objects on the primary surface; position/delta token families never collide
  with lane hues.
- **P7 — Tiers/bands reveal cliffs, legally.** Dividers/shaded groups keyed to thresholds;
  neutral labels legal today, calibrated-lexicon names gated behind the 00 amendment; no
  verdict hue.
- **P8 — Trend is a signed delta over a named window, never urgency.** Glyph-first,
  color-second, CVD-safe; direction AND basis; sparklines weighted so magnitude is legible.
- **P9 — Rankings are the primitive; every surface composes from it** (player card, Trade
  Lab, league, waiver, rookie board all reuse the row grammar).
- **P10 — Mobile is a first-class layout**, not a squeezed desktop (daily tape + top changed
  rows, status pill, bottom-sheet inspector, no overflow).
- **P11 — The unanchored visual-audit gate is standing governance.** Whole-viewport (not
  diff) is the review unit; scored benchmark-delta is the instrument; fresh-agent unanchored
  review is the standing pre-David gate; mid-scroll captures mandatory; contract-green is
  explicitly insufficient for a visual GREEN.
- **P12 — Density and cadence are habit mechanics.** 32px rows stay (fantasy-native); refresh
  cadence stated as a designed clock element; the overnight-movement story owns the hero.

**Palette caveat:** Sleeper position hues (QB pink/red, RB teal, WR blue, TE orange) are a
candidate *family* — the hue family is high-confidence, exact hexes must not enter the token
spec without direct screenshot sampling; position hues apply as chips/rings only, never
row-fills, and never collide with the blue/amber lane axes or delta colors.

## Enforceability (Codex): objective blockers + scored human gates

Split "best-in-class" into two enforceable tiers rather than vibes:

- **Objective blockers (real automated tests):** the scaffolding-hide law as a DOM/screenshot
  audit on user routes (fails on visible raw schema tokens/IDs/ISO/diagnostics outside
  approved receipt/drawer/dev surfaces); a viewport-first product test (first screen is a
  fantasy/asset story, not system status); receipt containment; benchmark component grammar
  (AssetRow primitives present on rankings/daily-open surfaces); no naked diagnostics in
  primary regions.
- **"Jaw-dropping" = mandatory independent fresh-agent scored audits** with a rubric
  (first-viewport story, fantasy-native identity, hierarchy, density, color discipline,
  mobile integrity, benchmark parity), thresholded, two unanchored passes, no P0/P1 — not
  brittle aesthetic snapshots.

## Constraint-integrity risks to pre-empt (Codex + Gemini)

1. **Aspiration language smuggles verdicts** — "edge / opportunity / target / priority /
   Elite / Bust" can become recommendations. Stay strictly descriptive.
2. **Rich color weakens lane isolation** or reintroduces verdict green/red. Content-is-style:
   team colors as micro-accents only; blue/amber own the lanes; weight/size is the primary
   contrast lever.
3. **Fantasy-native presentation over-trusts market data** — player cards/rankings must not
   look decision-grade before validation; market stays overlay-only.
4. **Hiding scaffolding must not hide caveats** — caveats become manager prose, not
   disappearance.
5. **Human visual gates rubber-stamp** unless kept unanchored, rubric-bound, failure-seeking.
